# 执行流程

收到任务后严格按以下流程执行，不许跳步、不许自行篡改顺序：

```
Phase 0 ─ 先对齐，再动手
├── 读任务 → 判断是否需要盘问
│   ├── 模糊/多解/边界不清 → 走盘问协议（目标/边界/优先级/成功标准）
│   └── 明确 → 直接进 Phase 1
├── 有问题一次性问完，别干到一半来问我
└── 用户新增/修改需求 → 回到 Phase 0 重新对齐

Phase 1 ─ 调研
├── docs/design.md → 回顾旧设计
├── 读代码 → 找到所有 需要修改的部分 + 受影响的代码（影响分析）
└── 盘点可用技能和工具

Phase 2 ─ 脑暴与决策（如有疑惑/矛盾/多解）
├── expert-brainstorm → 对复杂/模糊点执行脑暴+找茬循环
├── 解决所有 🔴/🟡 茬后再推进
└── 无疑惑直接跳 Phase 3

Phase 3 ─ grill-me 和我对细节
├── 调用 grill-me 和我对细节，把模糊点敲死
├── 所有疑问一次问完，禁止干到一半再来问
└── 确认完进 Phase 4

Phase 4 ─ 编码实现
├── 精准修改，只改必须改的部分
└── 禁止使用 build.bat 打包

Phase 5 ─ 验证
├── 先验证关键步骤能否跑通
├── 同一路径失败 ≥ 2 次 → 换方案（事不过三）
└── 验证通过 → 进 Phase 6

Phase 6 ─ 更新文档
├── docs/design.md → 写入详细功能设计（不是日志，不写代码）
└── docs/edits.md → 修改完成状态

Phase 7 ─ 交付
├── 打包 → adb 安装
├── 修改本文件的完成状态
└── 标记 [x] = 你完成了但我还没验收，保留不动
```

# 强制执行规则

1. 编码前：先读取设计文档
   接收任何编码任务（新增 / 修改功能、修复 bug），必须优先读取 docs\design.md 文档。
   需完整理解文档中所有已实现的功能设计，确保本次修改兼容现有设计，不破坏、覆盖旧功能，不凭空修改或遗忘旧逻辑。
2. 编码后：同步更新设计文档
   完成编码后，必须立刻同步更新 docs\design.md 文档，这是最高优先级的强制要求，不可跳过。
   更新规则如下：
   若文档不存在：自动创建文档，将本次实现的功能整理为规范的功能设计内容写入。
   若文档已存在：
   绝对不能删除未涉及本次修改的任何内容，完整保留所有已实现的旧功能设计。
   针对本次修改精准更新：
   新增功能：添加完整设计，包含功能名称、描述、交互逻辑、输入输出规则等核心信息。
   修改功能：定位对应旧功能条目，将过时内容完全替换为本次修改后的最新设计。
   删除功能：移除对应功能条目，或标记为「已废弃」并标注废弃时间。
3. 文档结构规范
   docs\design.md 需遵循以下层级结构，新增功能必须按此归类：
   # 界面：用户可见的 UI 功能，子分类：聊天与会话、会话与群组、智能体配置、模型与连接、技能与插件、记忆与洞察、安全与审批、数据与存储、应用设置、调试与诊断、引导
   # 系统：后台逻辑与引擎，子分类：运行时引擎、模型与路由、工具系统、多智能体协作、任务与流程、记忆与人格、通信与网络、存储与数据
   标题层级：# 大类 → ## 子分类 → ### 功能名称（无需添加【功能 x】前缀）
   功能条目格式：功能描述 / 交互逻辑 / 输入规则 / 输出结果

# 补充规则

| # | 规则 |
|---|------|
| 1 | 无法直接从代码上定位, 要功能运行日志 → 直接说(也可以在下方修改项前标注(需要日志))，我给你开 |
| 2 | 用户中途新增/修改需求 → 看到后自行走 Phase 0 重新对齐，不用等我催 |
| 3 | 优先级自己排，从高到低逐个一次性改完，别改一个来问一句也别来问我你该干哪个 |
| 4 | 被标记为 `[x]` 的项 → 我还没验收，保持不动 |
| 5 | 错误恢复：操作失败 → 停 → 回滚/评估 → 记教训 → 重试/换方案 → 多次失败告诉我 |
| 6 | 可以在我的修改项说明后面加修改方式, 但不要直接改我写的修改项说明! |
| 7 | 建立 修改-审查循环, 代码完成后要用 evo-code-review 进行审查并根据 问题、建议改进、提示 继续修改并审查, 直到审查不出问题为止 |
| 7.1 | 审查要重点注意性能问题(GC, 内存泄漏, 高耗时操作, 代码复用) 以及新增的功能是否被注册(死代码) |
| 8 | 所有修改项都完成后打包成apk(不用打包release), 禁止使用build_and_install进行打包 |
